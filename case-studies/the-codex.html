Case study

The Codex

The Codex is a live offline-first encyclopaedia of human technology built around dependency chains, documentary entries, and planner workflows. It is designed for users who need more than inspiration: they need practical structure, verification context, and a path from primitive capability to advanced systems.

Problem

Human technical knowledge is fragmented across videos, blogs, textbooks, and institutional silos. Most resources explain isolated topics but do not show prerequisites, materials, unlocks, hazards, or how one capability enables another.

The Codex addresses that gap by treating technologies as linked systems instead of disconnected articles.

Who it is for

  • People exploring collapse-readiness and long-horizon resilience.
  • Users who want a structured map of technology dependencies.
  • Planners interested in off-world or constrained-environment capability building.

Approach

  • Use a strict JSON data model for technology entries instead of hard-coding content into the UI.
  • Separate content validation from presentation so contributors can improve data without rewriting React code.
  • Build planner, pathfinder, search, and status views around the same core technology graph.
  • Make the app installable and offline-capable through a generated service worker and PWA setup.
  • Treat verification as product surface area, not just documentation afterthought.

What ships today

  • Interactive technology tree and detail pages.
  • Planner mode and pathfinder navigation.
  • Status dashboard for coverage, maturity, hazards, and media.
  • Obsidian export and offline caching behavior.
  • Public GitHub Pages deployment.

Tradeoffs

  • Trust burden is high because mistakes in safety-heavy domains matter.
  • Content depth and verification quality require sustained editorial discipline.
  • Visual polish and data architecture are ahead of expert-reviewed content depth in some branches.

Architecture preview used as proof artifact in the portfolio.

Outcome

The project is now a live flagship that proves structured-content modelling, offline-first product thinking, multi-view React application design, and the ability to build around a serious thesis instead of a tutorial pattern. It anchors the portfolio because it shows both engineering execution and product judgement.
